## Инструкция по интеграции промокод-системы для клиента

### Базовый URL
```
https://ваш-домен.com/api
```

### Аутентификация
Все запросы к промо-эндпоинтам требуют JWT-токен в заголовке:
```
Authorization: Bearer <токен-пользователя>
```
Токен получается при авторизации пользователя в приложении. Из токена автоматически извлекается `userId` — передавать его в теле запроса не нужно.

---

### Эндпоинт 1: Проверка промокода

**Когда вызывать:** пользователь ввёл промокод и нажал "Применить"

```
POST /api/promo/check
Content-Type: application/json
Authorization: Bearer <токен>

{
  "code": "TESTCODE"
}
```

**Ответы:**

| Ситуация | HTTP | Ответ |
|---|---|---|
| Код блогера | 200 | `{"type": "blogger", "bloggerName": "Имя блогера", "message": "Промокод блогера Имя блогера применён"}` |
| Премиум-код | 200 | `{"type": "premium", "durationDays": 7, "expiresAt": "2026-06-05T...", "message": "Премиум активирован на 7 дней"}` |
| Код не указан | 400 | `{"error": "code is required"}` |
| Премиум уже использован | 410 | `{"error": "Промокод уже использован"}` |
| Код не найден | 404 | `{"error": "Промокод не найден"}` |
| Сервис не настроен | 503 | `{"error": "Promo service not configured"}` |

**Логика на клиенте:**
- Если `type === "premium"` — подписка уже активирована на сервере, показать сообщение из `message`, обновить статус подписки в приложении (дата окончания в `expiresAt`)
- Если `type === "blogger"` — показать сообщение из `message`, запомнить код для отправки при покупке
- Если ошибка — показать сообщение из `error`

---

### Эндпоинт 2: Фиксация покупки

**Когда вызывать:** пользователь оплатил подписку и у него был применён промокод блогера

```
POST /api/promo/purchase
Content-Type: application/json
Authorization: Bearer <токен>

{
  "code": "TESTCODE"
}
```

**Ответ:** `200 {"success": true}`

> Вызывать **только** для блогерских промокодов (type === "blogger"). Для премиум-кодов вызывать не нужно — они обрабатываются полностью в `/check`.

---

### Пример потока

```
1. Пользователь авторизован, имеет JWT-токен
2. Пользователь вводит код "SALE2026"
3. Клиент -> POST /api/promo/check {code: "SALE2026"} с Authorization: Bearer <токен>
4. Сервер -> {type: "blogger", bloggerName: "Иван", message: "Промокод блогера Иван применён"}
5. Клиент запоминает код, показывает сообщение
6. Пользователь покупает подписку, оплата проходит
7. Клиент -> POST /api/promo/purchase {code: "SALE2026"} с Authorization: Bearer <токен>
8. Готово — в статистике блогера появится ввод + покупка
```

### Пример потока с премиум-кодом

```
1. Пользователь авторизован, имеет JWT-токен
2. Пользователь вводит код "PREMIUM7"
3. Клиент -> POST /api/promo/check {code: "PREMIUM7"} с Authorization: Bearer <токен>
4. Сервер -> {type: "premium", durationDays: 7, expiresAt: "2026-06-05T...", message: "Премиум активирован на 7 дней"}
5. Клиент обновляет статус подписки, показывает сообщение
6. Готово — подписка уже активна, вызывать /purchase НЕ нужно
```
